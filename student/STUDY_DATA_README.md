# 📁 学習データ管理について

## 💾 データ保存の仕組み

学習管理ページで学習時間を提出すると、**自動的にJSONファイルがダウンロード**されます。

### ファイル名の形式
```
{ユーザー名}_study_data.json
```

**例:**
- `tanaka_study_data.json`（田中さんの全期間のデータ）
- `suzuki_study_data.json`（鈴木さんの全期間のデータ）

**特徴:**
- ✅ 1ユーザー = 1ファイル（全ての週のデータを集約）
- ✅ 新しい週を提出すると、既存データに追加される
- ✅ 同じ週を再提出すると、その週のデータだけ更新される

## 📂 推奨フォルダ構造

ダウンロードしたファイルは、シンプルに保存できます：

```
study_data/
├── tanaka_study_data.json
├── suzuki_study_data.json
└── yamada_study_data.json
```

または、ユーザーごとにフォルダ分けも可能：

```
study_data/
├── tanaka/
│   └── study_data.json
├── suzuki/
│   └── study_data.json
└── yamada/
    └── study_data.json
```

## 📊 JSONファイルの内容（コンパクト形式）

各ファイルには複数週のデータが含まれます：

```json
{
  "2026-01-06": {
    "国語": 120,
    "数学": 150,
    "英語": 180,
    "理科": 90,
    "社会": 60,
    "課題": 100,
    "提出日時": "2026-01-12T15:30:00.000Z"
  },
  "2026-01-13": {
    "国語": 130,
    "数学": 140,
    "英語": 200,
    "理科": 80,
    "社会": 70,
    "課題": 90,
    "提出日時": "2026-01-19T14:20:00.000Z"
  }
}
```

- キー: 週の開始日（月曜日）
- 値: その週の学習時間データと提出日時

## 🔄 データの更新

同じ週のデータを再提出すると：
- ✅ 同じファイル名で新しいファイルがダウンロードされる
- ⚠️ ブラウザの設定により、既存ファイルに `(1)`, `(2)` などが追加される場合がある
- 💡 最新のファイルを使用し、古いファイルは削除することをお勧めします

## 💡 活用方法

### Excelでの分析
1. JSONファイルをテキストエディタで開く
2. データ部分をコピー
3. Excelに貼り付けてグラフ化

### 長期保存
- 定期的に `study_data` フォルダをバックアップ
- Google DriveやOneDriveなどのクラウドに保存

### データの復元
- ファイルを保存しておけば、いつでも過去のデータを確認可能
- 必要に応じてlocalStorageにインポート可能（将来実装予定）

## ⚠️ 注意事項

1. **ファイルは自動的には整理されません** - 手動で整理する必要があります
2. **ダウンロードフォルダを定期的に確認** - ファイルが溜まりすぎないように
3. **バックアップを忘れずに** - 大切なデータは複数箇所に保存
